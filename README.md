产品需求文档 (PRD) & 技术设计文档 (TDD):
Cognitive Zen Breathing Reader
1. 产品概述 (Product Overview)
项目名称： Cognitive Zen Breathing Reader (认知禅意呼吸阅读器)
核心愿景： 将长文阅读从一种沉重的“认知负担”转化为一种“感官流感”、“深度理解”、“数字冥想”的艺术体验。
核心价值： 消除阅读焦虑，建立深度认知，实现长文阅读的“心流状态”，提升阅读的幸福感。
1.1 问题定义
- 长文排版焦虑: 密集的纯文本导致视觉疲劳和注意力涣散。
- 认知壁垒: 晦涩的专业术语和逻辑让读者难以建立直观理解。
- 摘要局限性: 传统的 AI 摘要虽然精简,但往往丢失了原作者的论证美感与逻辑张力。

1.2 目标受众
深度学习者： 需要消化高难度论文、长篇深度报道的专业人士。
注意力缺失人群： 难以长时间在密集文本上保持专注的读者。
数字游民/冥想爱好者： 追求高审美、极简主义交互体验的用户。

1.3 核心解决方案
结合 "认知转换" (Cognitive Transformation) 与 "动态排版" (Dynamic Typography),通过 AI
将深奥内容重构为多层级类比,并利用视觉节奏同步读者的呼吸与专注力。

2. 核心功能需求 (Functional Requirements)
2.1 认知重构模块 (Zen Rewriter)
- 多层级类比转换: 用户可选 5 个维度的转换模式:
- 童话隐喻 (5岁级别)
- 现实生活映射 (常识级别)
- 叙事长诗 (情感与逻辑结合)
- 跨学科类比 (如用生物学解释互联网架构)
- 禅意公案 (极简抽象)
- 保真切换: 支持在原声与重构内容之间快速切换,点击特定段落查看原文对照。
2.2 呼吸排版引擎 (Breathing UI)
- 逻辑热图: AI 自动分析文章段落权重。
- 核心论点: 字体略微加粗,伴随极慢频率的呼吸律动(Scale 1.0 -> 1.02)。
- 支撑论据: 略微半透明,字间距舒展。
- 情感激昂处: 文字带有微弱的边缘光晕,伴随极其轻微的抖动。
- 自适应舒展: 遇到复杂度极高的句子(长难句),系统自动增加行高和字间距,为大脑腾出
“思考空间”。
- 眼动追踪同步 (Eye Tracking Sync): 基于摄像头进行无感知眼动追踪,实时检测用户的专注状态、眨眼频率与注视模式,根据眼动数据自动调节呼吸排版的频率与振幅,实现阅读节奏与生理状态的深度适配。

3. 设计规范 (Design Specifications)
3.1 视觉风格
- 色调: 采用保护视力的莫兰迪色系或纸张纹理感背景。
- 字体: 衬线体(用于沉浸式阅读)与非衬线体(用于重构类比)的平滑切换。
- 动效: 所有动效必须遵循 ease-in-out 曲线,频率控制在 0.1Hz - 0.2Hz(成年人静息呼
吸频率)。
3.2 交互逻辑
- 专注模式: 隐藏所有边框,文字居中。
- 节奏控制器: 用户可手动调节“呼吸速率”,系统根据滚动速度自动微调。

4. 技术架构设计 (Technical Design)
4.1 后端:AI 认知处理层
- 模型方案: 建议使用 GPT-4o 或 Claude 3.5 Sonnet(后者在长文理解和文学重构上表现
更优)。
- Prompt 策略: 采用 Chain-of-Thought (CoT) 提取文章的底层逻辑树,再根据选定模板进
行转换。
4.2 前端:动态排版引擎
- 文本解析: 使用 NLP.js 或类似库对句子复杂度进行初步评估。
- 动画实现:
- 使用 CSS Variables 控制 letter-spacing 和 font-weight。
- 使用 RequestAnimationFrame 确保平滑度。
- Intersection Observer API: 仅对当前视口内的段落执行动画,以节省性能。
- 眼动追踪技术: 采用 MediaPipe Iris 进行实时虹膜定位与追踪,检测眨眼频率、注视时长、眼动轨迹等指标。所有摄像头数据仅在本地处理,不进行任何上传,确保用户隐私安全。根据眼动模式判断用户专注度,动态调整呼吸排版参数。

4.3 浏览器插件实现 (Chrome Extension)
- Content Script: 注入阅读层覆盖原网页。
- Readability.js: 预处理网页,提取干净的正文内容。

5. 风险与挑战 (Risk & Constraints)
1. 认知偏差: AI 转换可能导致原意曲解。需增加”置信度”显示。
2. 性能开销: 复杂的 CSS 动效在低配设备上可能卡顿。需提供”低功耗/静态模式”。
3. 晕动症: 动态跳动需非常隐晦,避免用户产生生理不适。
4. 隐私保护: 眼动追踪数据(摄像头采集)严格限制在本地处理,不得上传至任何服务器,用户可随时关闭此功能。

6. 路线图 (Roadmap)
- Phase 1 (MVP): 实现基本的内容类比重构 + 静态的“舒展”排版。
- Phase 2 (Beta): 加入动态呼吸动效与滚动速度同步。
- Phase 3 (Final): 多端同步与社区“禅意阅读”笔记分享。
